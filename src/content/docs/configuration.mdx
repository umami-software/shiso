Shiso uses two configuration layers:

- `shiso.config.json` for runtime setup
- `docs.json` for docs navigation structure

## `shiso.config.json` (app config)

```json
{
  "contentDir": "./src/content",
  "docsConfigPath": "./src/docs.json",
  "docs": {
    "title": "Docs - Shiso"
  }
}
```

### Supported keys

- `contentDir` (required): root folder containing section folders such as `docs`
- `docsConfigPath` (optional): path to docs config file; defaults to `./docs.json`
- `docs.title` (optional): used as metadata title suffix
- `docs.top` (optional): top offset for sticky docs side panels

### Conformance review: current app

`src/shiso.config.json` conforms to the supported shape:

```json
{
  "contentDir": "./src/content",
  "docsConfigPath": "./src/docs.json",
  "docs": {
    "title": "Docs - Shiso"
  }
}
```

Notes:

- `contentDir` points at `./src/content`
- `docsConfigPath` points at `./src/docs.json`
- `docs.title` is present and valid

## `docs.json` (Mintlify-style navigation config)

This app consumes Mintlify-style navigation and resolves docs pages from file slugs.

```json
{
  "$schema": "https://mintlify.com/docs.json",
  "name": "Shiso",
  "navigation": {
    "tabs": [
      {
        "tab": "Documentation",
        "groups": [
          {
            "group": "Getting started",
            "pages": ["index", "installation", "configuration"]
          },
          {
            "group": "Writing content",
            "pages": ["writing-content", "text", "images", "lists", "code-blocks"]
          }
        ]
      },
      {
        "tab": "Components",
        "groups": [
          {
            "group": "Components",
            "pages": [
              "components/index",
              "components/accordion",
              "components/callout"
            ]
          }
        ]
      }
    ]
  }
}
```

### Page reference mapping

Each `pages` entry maps to a file under `contentDir/docs`:

- `"index"` -> `src/content/docs/index.mdx` (or `.md`)
- `"installation"` -> `src/content/docs/installation.mdx`
- `"components/index"` -> `src/content/docs/components/index.mdx`

Routes are generated as:

- `index` -> `/docs`
- `components/index` -> `/docs/components`
- `components/callout` -> `/docs/components/callout`

## What this app uses from `docs.json`

- navigation hierarchy for docs pages
- tab labels (for top nav)
- group labels (for side nav sections)
- page order (for prev/next)

For full shape details and unsupported entries, see [docs.json support](/docs/docs-json-support).
