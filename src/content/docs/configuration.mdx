---
title: Configuration
---

Shiso uses a single configuration file: `shiso.config.json`.
That file uses the Mintlify `docs.json` schema directly.

## Example

```json
{
  "$schema": "https://mintlify.com/docs.json",
  "theme": "mint",
  "name": "Shiso",
  "navigation": {
    "tabs": [
      {
        "tab": "Documentation",
        "groups": [{ "group": "Getting started", "pages": ["index", "installation"] }]
      }
    ]
  }
}
```

## Supported keys

### Mintlify schema keys used in this app

- `$schema`
- `theme`
- `name`
- `navigation`

## Current app conformance

`src/shiso.config.json` is the source of truth for navigation and conforms to this model:

- it includes Mintlify navigation schema fields directly
- no separate `docs.json` file is required

## Page reference mapping

Each page slug in `navigation` maps to a file under `src/content/docs`:

- `"index"` -> `src/content/docs/index.mdx` (or `.md`)
- `"installation"` -> `src/content/docs/installation.mdx`
- `"components/index"` -> `src/content/docs/components/index.mdx`

Routes are generated as:

- `index` -> `/docs`
- `components/index` -> `/docs/components`
- `components/callout` -> `/docs/components/callout`

For supported navigation shapes and limits, see [docs.json support](/docs/docs-json-support).
