---
title: Installation
---

This guide wires Shiso docs into a Next.js App Router project.

## Prerequisites

- Next.js App Router project
- A `src/content/docs` folder (or equivalent) for docs files

## 1. Install Shiso

```shell
npm install @umami/shiso
```

## 2. Create a docs route

Add a catch-all route:

```text
src
├── app
│   └── docs
│       └── [[...slug]]
│           └── page.jsx
```

`page.jsx`:

```js
import { Shiso } from '@umami/shiso';
import { next } from '@umami/shiso/server';
import config from 'path/to/shiso.config.json';

const { generateMetadata, generateStaticParams, renderPage } = next('docs', config);

export { generateMetadata, generateStaticParams };

export default renderPage(props => {
  return <Shiso {...props} />;
});
```

## 3. Add `shiso.config.json`

Point Shiso at your content and docs config:

```json
{
  "contentDir": "./src/content",
  "docsConfigPath": "./src/docs.json",
  "docs": {
    "title": "Docs - My App"
  }
}
```

## 4. Add `docs.json`

Define navigation with Mintlify-style structure:

```json
{
  "$schema": "https://mintlify.com/docs.json",
  "name": "My Docs",
  "navigation": {
    "tabs": [
      {
        "tab": "Documentation",
        "groups": [
          {
            "group": "Getting started",
            "pages": ["index", "installation"]
          }
        ]
      }
    ]
  }
}
```

## 5. Create matching content files

Each page slug in `docs.json` must resolve to a file under `contentDir/docs`:

```text
src/content/docs/index.mdx
src/content/docs/installation.mdx
```

## 6. Run locally

```shell
npm run dev
```

Then open `/docs` and confirm:

1. docs page renders
2. side navigation matches `docs.json`
3. prev/next links follow your page order
4. heading anchors appear in "On this page"
