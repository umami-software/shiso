Shiso in this app is an MDX docs renderer for Next.js App Router. It reads Mintlify-style navigation
schema from `shiso.config.json`, resolves content files, compiles MDX, and renders tab/group navigation.

## Supported today

### Content and rendering

- `.mdx` and `.md` files in `src/content/docs`
- frontmatter fields such as `title` and `description`
- GitHub-flavored Markdown via `remark-gfm`
- heading IDs and on-page anchors
- syntax highlighting via `rehype-highlight`
- Zen components directly in MDX (`@umami/react-zen`)

### Navigation from `shiso.config.json`

- `navigation.tabs`
- `navigation.dropdowns`
- `navigation.groups` + `navigation.pages`
- `navigation.versions` (chooses `default: true`, otherwise first)
- `navigation.languages` (chooses `default: true`, otherwise first)
- nested groups inside `pages`
- prev/next links from normalized page order

### Built-in docs components

This app ships a Mintlify-style MDX component set:

- `Accordion`, `AccordionGroup`
- `Callout`, `Note`, `Tip`, `Warning`, `Info`, `Check`
- `Card`, `CardGroup`
- `CodeGroup`
- `Expandable`
- `Frame`
- `Param`, `ParamField`
- `RequestExample`
- `ResponseField`
- `Tabs`, `Tab`
- `Steps`, `Step`
- `Tooltip`

## Out of scope in this app

This implementation does not attempt to reproduce Mintlify hosted platform features (for example hosted
theme presets, analytics integrations, or OpenAPI-driven reference generation). It focuses on local docs
rendering and navigation.
